## 📖 1. مقدمة عن المشروع
**"شحنتي"** هي منصة لوجستية ذكية تهدف إلى إعادة صياغة تجربة الشحن والنقل البري في مصر. تعمل المنصة كحلقة وصل تقنية متطورة تربط بين التجار (أصحاب البضائع)، السائقين (أصحاب المركبات)، وشركات الشحن، مع توفير رقابة حكومية وإدارية دقيقة للمحافظات والمسؤولين.

تتميز المنصة بتجربة مستخدم (UX) فاخرة تعتمد على أحدث تقنيات الويب، مع التركيز على السرعة، الأمان، والشفافية في نظام المزايدات وتتبع الشحنات.

---

## 🎯 2. الأهداف والرؤية
- **الرقمنة الكاملة**: استبدال العمليات الورقية التقليدية بنظام إلكتروني متكامل.
- **الشفافية**: توفير نظام مزايدة حر يضمن أفضل سعر للتاجر وأفضل ربح للسائق.
- **الأمان**: تتبع لحظي للشحنات والتحقق من هوية السائقين والمركبات.
- **دعم السيادة**: تسهيل تحصيل الرسوم السيادية للمحافظات بدقة وسرعة.

---

## 👥 3. أدوار المستخدمين ومنطق العمل (User Personas)

| الدور | الوظائف الرئيسية |
| :--- | :--- |
| **التاجر (Customer)** | إنشاء شحنات، اختيار أفضل العروض، تتبع البضائع، تقييم السائقين. |
| **السائق (Driver)** | تقديم عروض أسعار، إدارة الرحلات، الإبلاغ عن تنبيهات الطريق، متابعة الأرباح. |
| **شركة الشحن (Company)** | إدارة أسطول من المركبات والسائقين، مراقبة العمليات الكبرى، إدارة العقود. |
| **المحافظة (Governorate)** | مراقبة حركة النقل في النطاق الجغرافي، التحقق من إيصالات الرسوم، التقارير. |
| **المسؤول (Admin)** | الرقابة العامة، فض النزاعات، إدارة إعدادات النظام، تفعيل الحسابات. |

## ✨ 4. المميزات التقنية الرئيسية

### 📦 نظام إدارة الشحنات الذكي
- دورة حياة كاملة للشحنة من (Pending -> Bidding -> In Transit -> Delivered).
- حساب تلقائي للأوزان، الأبعاد، والرسوم التقديرية.

### 💰 نظام المزايدة الحي (Real-time Bidding)
- يسمح للسائقين بتقديم عروض سعرية تنافسية.
- لوحة تحكم للتاجر للمقارنة بين العروض بناءً على السعر، التقييم، ونوع المركبة.

### 📍 التتبع والخرائط
- تتبع لحظي لموقع الشحنة باستخدام خرائط تفاعلية.
- تحديثات الحالة التلقائية عند الوصول لنقاط العلام (Checkpoints).

### 🚨 نظام تنبيهات الطريق الميداني
- إمكانية الإبلاغ عن (حوادث، زحام، إغلاق طرق) لتنبيه السائقين الآخرين في نفس المسار.

### 📄 التوثيق والعقود الرقمية
- توليد عقود قانونية إلكترونية فور قبول العرض لضمان حقوق كافة الأطراف.

## 🛠️ 5. البنية التكنولوجية (Tech Stack)

تم بناء المنصة باستخدام أحدث الأدوات لضمان أفضل أداء وقابلية للتوسع:

- **Frontend**: `React 19` مع `Vite` لسرعة التطوير والبناء.
- **Styling**: `Tailwind CSS 4` مع لمسات جمالية باستخدام `Framer Motion` للأنيميشن.
- **State Management**: `Redux Toolkit` لإدارة الحالة المعقدة وتدفق البيانات.
- **Routing**: `React Router 7` لإدارة المسارات والحماية (Protected Routes).
- **Icons & UI**: `Lucide React` للأيقونات، ومكونات مخصصة مبنية على `Radix UI` و `MUI`.
- **Forms & Validation**: `React Hook Form` مع `Zod` لضمان صحة البيانات المدخلة.
- **Real-time**: دعم `Socket.io-client` للتحديثات اللحظية.

## 📂 6. هيكل المشروع (Project Structure)

```text
src/
├── api/                # إعدادات Axios والروابط الأساسية للمدخلات
├── components/         # المكونات المشتركة (Buttons, Cards, Modals)
│   └── ui/             # مكونات واجهة المستخدم الذرية (Atomic UI)
├── features/           # الميزات مقسمة حسب الدور الوظيفي
│   ├── auth/           # نظام التسجيل والدخول والتحقق
│   ├── admin/          # لوحة تحكم الإدارة العليا
│   ├── bidding/        # منطق المزايدات والعروض
│   ├── driver/         # واجهات السائق والرحلات
│   ├── shipments/      # إدارة الشحنات والبيانات
│   └── tracking/       # خرائط وتتبع الشحنات
├── store/              # مخزن الحالة (Redux Slices & Custom Hooks)
├── layouts/            # قوالب الصفحات (Main Layout, Dashboard Layout)
├── routes/             # تعريف كافة المسارات وصلاحياتها
└── utils/              # الدوال المساعدة (تنسيق التاريخ، التعامل مع النصوص)
```

## 🚀 7. تشغيل المشروع (Installation)

1. **تثبيت التبعيات**:
   ```bash
   npm install
   ```

2. **إعداد المتغيرات البيئية**:
   قم بإنشاء ملف `.env` وإضافة الرابط الأساسي للـ API:
   ```env
   VITE_API_URL=your_api_endpoint
   ```

3. **بدء التطوير**:
   ```bash
   npm run dev
   ```

4. **بناء نسخة الإنتاج**:
   ```bash
   npm run build
   ```
